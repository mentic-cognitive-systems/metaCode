# Application Architectures (WHAM and AXIS)

We have been exploring different application architectures to find the best way to structure our code and manage complexity. We started with "WHAM" which was built by breaking the classic MVC architecture into static and dynamic parts, but we found that it still had some issues with separation of concerns and clarity. After further iteration, we evolved our thinking into the "AXIS" architecture, which has clearer roles for each component and better separation of concerns.

Both architectures are documented here for reference, but we recommend using AXIS for new projects as it is more robust and intuitive. The WHAM architecture is included to show the evolution of our thinking and to provide context for the design decisions we made in AXIS.
